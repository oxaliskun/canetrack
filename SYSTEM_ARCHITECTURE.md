# CaneTrack System Structure & Flow

## Architecture Overview
- **Frontend**: React 19 + Vite + Tailwind CSS + Framer Motion + Sonner (for toasts)
- **Backend**: Express.js
- **Database**: SQLite + Prisma ORM
- **Authentication**: JWT Based

## Models
1. **User**: Represents system users (Farmers, Operators, Receivers, Admins).
2. **Farm**: Belongs to Farmers. Stores farm metrics and serves as origin for sugarcane.
3. **WeightTicket**: Core entity. Tracks sugarcane lifecycle from arrival to reconciliation.
4. **ReconciliationRecord**: Represents the discrepancy review generated by Receivers.
5. **AuditLog**: Traces all system-critical operations.
6. **Notification/SystemSettings**: Added to handle global configurations and alerts natively.

## System Flow (Role-Based)
### 1. The Operator (Mill Entry)
- Logs into `OperatorDashboard`.
- Fills out truck arrival data: *Truck Plate, Farm Origin, Gross Weight, Tare Weight*.
- System computes: `Mill Weight = Gross - Tare`.
- Generates a **Pending** ticket.
- *Enhancement*: Operator can now print out physical receipts right from the dashboard.

### 2. The Receiver (Refinery Validation)
- Logs into `ReceiverDashboard`.
- Views the queue of `PENDING` tickets arriving at the refinery.
- Inputs the newly measured `Refinery Weight`.
- System computes variance. If `abs(Mill Weight - Refinery Weight) > 50kg`, the ticket is marked **DISPUTED**. Otherwise, it becomes **RECONCILED**.
- *Enhancement*: Live variance previews are shown before ticket submission natively in the UI.

### 3. The Admin (Command Center)
- Logs into `AdminDashboard`.
- Global overview of financial flow and discrepancies.
- Can review the `Audit & Reconciliation Logs`. 
- Can **Resolve** disputed tickets by reviewing the variance and appending correction notes.
- Capable of creating profiles for Operators/Receivers via internal provision forms.

### 4. The Farmer (Client Side)
- Logs into `FarmerDashboard`.
- Tracks estimated active payouts relying on their accumulated delivery `totalValue` and total `millWeight`.
- View chronological list of `WeightTicket` occurrences linked directly to their farmlands.

## Important Mechanics
- JWT Token Validation occurs upon every backend request.
- The platform uses dynamic styling and smooth layouts with Framer Motion (e.g., Live Variance indicator in receiver's dashboard).
- State/Data changes execute strict validations via Prisma ORM preventing phantom tickets.
